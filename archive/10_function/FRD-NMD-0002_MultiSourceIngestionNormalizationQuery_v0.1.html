1. Overview

Summary: The ING module handles ingestion of reference data and daily updates from multiple external source types, generates derived snapshots, normalizes data against the product canonical, and exposes query/export APIs for downstream consumption. It is the operational core of NMD — everything between source and consumer.

Purpose: This document specifies the functional requirements for the Multi-Source Ingestion, Normalization & Query module within the Narwhal Market Data platform. It translates PC-NMD-0001 requirements FR-002 through FR-008 into detailed, testable specifications.

Audience: Developers, QA engineers, data owners, and downstream system integrators.

2. Scope

In Scope

  1. Reference data ingestion from external sources (FR-002)
  2. Daily update ingestion — settlement prices, margin rates, corporate events (FR-003)
  3. Multi-source type support — exchange websites, public APIs, SFTP, structured documents (CSV, XML, PDF), data feeds (FR-004)
  4. Derived snapshot generation from multiple external sources (FR-005)
  5. Data normalization and clean-up linked to product canonical (FR-006)
  6. Centralized ingestion framework to manage and track all source processing (FR-007)
  7. Query and export APIs for downstream system consumption (FR-008)

Out of Scope

  1. Product canonical maintenance (covered by FRD-NMD-0001)
  2. UI for data owners to manage sources, ingestion, and data (FR-009)
  3. Onboarding new data feeds via common framework without core changes (FR-010)
  4. Real-time streaming and subscription-based data delivery (Phase 3)
  5. Direct integration or data delivery to downstream systems (Phase 2)
  6. Equities (Phase 2)
  7. Fixed income and structured products (Phase 3)

3. Definitions

Term Definition
Snapshot A point-in-time extract of data from one or more sources, used as input to the normalization pipeline.
Derived Snapshot A snapshot generated by combining and transforming data from multiple source snapshots, constructed as another source by parsing source data in a configured step sequence.
Ingestion Framework The centralized component that manages, schedules, tracks, and monitors the processing of all data sources.
Source Adapter A pluggable component that handles connectivity and parsing for a specific source type (e.g., SFTP, REST API, website scraper).
Active Layer Storage tier containing the latest reference data and latest daily update (date and version) per instrument. This is the data served by the query API.
Archive Layer Storage tier containing all previous versions and previous dates of data. Retained for audit and historical queries.

6. Functional Overview

The ING module operates as a pipeline with four stages: (1) Source Connectivity — adapters connect to external sources (SFTP, API, website, document, feed) and pull raw data; (2) Ingestion — raw data is parsed and stored as source snapshots, with reference data and daily updates handled separately; (3) Normalization — source snapshots are cleaned, transformed, and linked to product canonical records, and derived snapshots are generated from multiple sources; (4) Query/Export — normalized data is exposed via APIs for downstream consumption. The Ingestion Framework (FR-007) manages and tracks all stages.

9. Business Rules

ID Rule Affects
BRL-001 A source snapshot must be fully ingested before normalization begins — no partial processing FR-ING-014, FR-ING-015
BRL-002 Derived snapshots are constructed as another source by parsing source data in a configured step sequence — each step overlays the previous, so later steps override earlier values for the same field FR-ING-014
BRL-003 Settlement prices may be updated multiple times within the same day (corrections, restatements). Each update creates a new version. The system tracks all versions and tags the latest version as current FR-ING-010, FR-ING-013
BRL-004 Normalization must not create or modify product canonical records — only link to existing ones FR-ING-016
BRL-005 Ingestion runs for the same source must not execute concurrently — queue or reject if already running FR-ING-019, FR-ING-020, FR-ING-021
BRL-006 All API query results return data from the latest completed normalization run only — in-progress data is never exposed FR-ING-024, FR-ING-025, FR-ING-026
